## Why a wrapper, not the SDK directly?

The kit is a CLI built for a human. Lighter MCP is the safety harness for a **non-human caller**.
Every call is schema-validated, gate-checked, audited, and (for write paths) preview-confirmed
**before** any subprocess runs against the exchange. The agent literally cannot reach the kit
without going through these layers.

<CardGroup cols={2}>
  <Card title="Typed Pydantic schemas" icon="shield-check">
    Symbol regex blocks shell metacharacters and CLI flag injection (`--side=long`) at the schema layer.
  </Card>
  <Card title="Mode-based catalog" icon="layer-group">
    `readonly` → `paper` → `live` → `funds`. Higher-risk tools physically don't exist in the catalog until you opt in.
  </Card>
  <Card title="Atomic notional ledger" icon="coins">
    UTC-keyed daily counter persists across restarts. Fail-safes to *cap exhausted* if its file is corrupt.
  </Card>
  <Card title="Two-step confirmations" icon="key">
    First call returns a preview + single-use token bound to tool name + argument digest. TTL-limited.
  </Card>
</CardGroup>

## Modes at a glance

| Mode      | Reads | Paper | Live | Funds | Tools |
| --------- | :---: | :---: | :--: | :---: | :---: |
| `readonly`| ✅    | —     | —    | —     | 18    |
| `paper`   | ✅    | ✅    | —    | —     | 29    |
| `live`    | ✅    | ✅    | gated| —     | 37    |
| `funds`   | ✅    | ✅    | gated| gated | 39    |

`live` and `funds` additionally require explicit `[live]` / `[funds]`
config blocks with allowlists, notional caps, leverage caps, and
confirmation flags. See [Configuration](/get-started/configuration).

## Install in 3 commands

```bash
git clone https://github.com/elliottech/lighter-agent-kit ~/lighter-agent-kit
cd ~/lighter-agent-kit && bash install.sh

git clone https://github.com/iamgoatedaf/lighter-mcp
bash lighter-mcp/install.sh --kit-path ~/lighter-agent-kit
```

That gives you a working `readonly` server. Promote to `paper`, `live`,
or `funds` from the [Modes & safety](/get-started/modes-and-safety)
guide.
