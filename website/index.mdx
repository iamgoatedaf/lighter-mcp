This page will help you get started with Lighter MCP — a portable bridge
between any MCP-capable AI agent and the Lighter perpetuals exchange.

The server wraps the official **lighter-agent-kit** in three things the
kit does not have on its own:

1. A **mode-based catalog** (`readonly` → `paper` → `live` → `funds`) so
   higher-risk tools do not even exist until you opt in.
2. **Pydantic-validated schemas** that block shell-metacharacter and
   CLI-flag injection before any subprocess runs.
3. A **tamper-evident JSONL audit log** with sanitized arguments, mode
   tags, and result/error envelopes for every call.

## Install

The server itself is one Python package; it shells out to a separate
`lighter-agent-kit` checkout for the actual signing.

```bash Shell
git clone https://github.com/elliottech/lighter-agent-kit ~/lighter-agent-kit
cd ~/lighter-agent-kit && bash install.sh

git clone https://github.com/iamgoatedaf/lighter-mcp
bash lighter-mcp/install.sh --kit-path ~/lighter-agent-kit
```

By default `install.sh` writes a `~/.lighter-mcp/config.toml` with
`mode = "readonly"`. That gives you the 18 read-only and diagnostics
tools and never touches a private key.

If you want the simulator, live orders, or withdrawals, see
[Modes & safety](/get-started/modes-and-safety).

## Run

```bash Shell
lighter-mcp serve
```

The process speaks MCP over stdio and exposes a tool catalog that
depends on the configured mode. Connect it from your agent using one of
the [adapters](/adapters/overview) — there are pre-built setups for
Cursor, Claude Code, Claude Desktop, Codex, and OpenClaw, plus a
generic MCP client recipe.

## Modes at a glance

| Mode        | Reads | Paper | Live  | Funds | Tools |
| ----------- | :---: | :---: | :---: | :---: | :---: |
| `readonly`  |   ✓   |   —   |   —   |   —   |  18   |
| `paper`     |   ✓   |   ✓   |   —   |   —   |  29   |
| `live`      |   ✓   |   ✓   | gated |   —   |  37   |
| `funds`     |   ✓   |   ✓   | gated | gated |  39   |

`live` and `funds` additionally require explicit `[live]` / `[funds]`
config blocks with allowlists, notional caps, leverage caps, and
confirmation flags. See [Configuration](/get-started/configuration).

## Project status

<Note>
**Alpha (`0.1.0`).** The tool surface is stable enough to use, but
expect schema-level breaking changes between minor versions until
`1.0`. See the
[CHANGELOG](https://github.com/iamgoatedaf/lighter-mcp/blob/main/CHANGELOG.md).
</Note>

<Warning>
Live orders and withdrawals on Lighter are **irreversible**. Read the
[disclaimer](/security/disclaimer) before flipping `mode = "live"` or
`mode = "funds"`.
</Warning>
