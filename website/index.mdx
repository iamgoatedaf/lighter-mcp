This page will help you get started with Lighter MCP тАФ a portable bridge
between any MCP-capable AI agent and the Lighter perpetuals exchange.

The server wraps the official **lighter-agent-kit** in three things the
kit does not have on its own:

1. A **mode-based catalog** (`readonly` тЖТ `paper` тЖТ `live` тЖТ `funds`) so
   higher-risk tools do not even exist until you opt in.
2. **Pydantic-validated schemas** that block shell-metacharacter and
   CLI-flag injection before any subprocess runs.
3. A **tamper-evident JSONL audit log** with sanitized arguments, mode
   tags, and result/error envelopes for every call.

## Install

One command, end to end:

```bash
pipx install lighter-mcp && lighter-mcp init
```

`lighter-mcp init` clones the upstream `lighter-agent-kit`, writes a
default `~/.lighter/lighter-mcp/config.toml` in `readonly` mode, and
auto-wires every MCP-capable agent it finds locally тАФ Cursor, Claude
Code, Claude Desktop, Codex тАФ including slash-commands, the
`lighter-trader` sub-agent, and the post-trade hook. Restart your
agent and try `/lighter-status`.

Live trading stays OFF until you flip it on in the config and re-run
`lighter-mcp doctor`. See [Modes & safety](/get-started/modes-and-safety).

Alternative install paths (`uvx`, Docker, source) are documented on
the [Installation](/get-started/installation) page.

## Run

The MCP server is launched automatically by your agent over stdio.
For standalone or daemon use:

```bash
lighter-mcp stdio                                # default, used by agents
lighter-mcp serve --host 127.0.0.1 --port 8791   # streamable HTTP
```

Connect it from your agent using one of the
[adapters](/adapters/overview) тАФ pre-built setups for Cursor, Claude
Code, Claude Desktop, Codex, and OpenClaw, plus a generic MCP client
recipe.

## Modes at a glance

| Mode        | Reads | Paper | Live  | Funds | Tools |
| ----------- | :---: | :---: | :---: | :---: | :---: |
| `readonly`  |   тЬУ   |   тАФ   |   тАФ   |   тАФ   |  18   |
| `paper`     |   тЬУ   |   тЬУ   |   тАФ   |   тАФ   |  29   |

`live` and `funds` additionally require explicit `[live]` / `[funds]`
config blocks with allowlists, notional caps, leverage caps, and
confirmation flags. See [Configuration](/get-started/configuration).
