The Cursor adapter wires up:

- The MCP server in `~/.cursor/mcp.json` (or per-project `.cursor/mcp.json`).
- A Cursor rule (`.cursor/rules/lighter.mdc`) that primes the assistant on safe Lighter use.
- Slash commands `/lighter-status`, `/lighter-positions`, `/lighter-kill`, `/lighter-paper`, `/lighter-audit`.
- The `lighter-trader` sub-agent with a narrow tool budget.
- An optional pre-tool hook that blocks live writes if `lighter_safety_status` reports caps exhausted.

## Install

```bash
lighter-mcp init --agents cursor
```

That's it. The installer patches `~/.cursor/mcp.json`, drops scaffolds
into your home directory, and runs `lighter-mcp doctor`. To wire a
specific project instead of the user-global profile:

```bash
lighter-mcp init --agents cursor --project-dir /path/to/your/project
```

The installer is idempotent — re-run it after upgrades.

## What gets written

```
~/.cursor/
└── mcp.json                     # adds the "lighter" entry under mcpServers

~/.cursor/                       # or <project>/.cursor/ with --project-dir
├── rules/lighter-safety.mdc     # safety rule auto-loaded by Cursor
├── hooks.json                   # post-tool-call hook config
├── hooks/after-lighter-trade.sh # hook script (logs + macOS notify)
├── commands/lighter-*.md        # 5 slash commands
└── agents/lighter-trader.md     # sub-agent definition
```

## mcp.json entry

```json
{
  "mcpServers": {
    "lighter": {
      "command": "lighter-mcp",
      "args": ["stdio"],
      "env": {
        "LIGHTER_MCP_CONFIG": "/Users/you/.lighter/lighter-mcp/config.toml"
      }
    }
  }
}
```

The single env var the server needs is `LIGHTER_MCP_CONFIG` — the kit
path, mode, and safety caps all live inside that TOML file. If
`lighter-mcp` isn't on `PATH` (rare with `pipx install`), use the
absolute path returned by `which lighter-mcp` as `command`.

## Reload after config changes

`Cmd-Shift-P` → **Reload MCP servers** (or just restart Cursor).

## Verifying

After Cursor reloads, open the chat panel and run:

```
/lighter-status
```

You should see the active mode, daily-notional remaining, equity, and
open orders. If the slash command isn't recognised, re-run
`lighter-mcp init --agents cursor` with `--project-dir` pointing at the
workspace where you opened Cursor.
