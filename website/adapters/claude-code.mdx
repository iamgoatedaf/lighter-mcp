The Claude Code adapter ships:

- An MCP entry in `~/.claude/mcp.json`.
- The shared slash commands and `lighter-trader` sub-agent.
- A post-tool-call hook for the audit notifications log.

## Install

```bash
lighter-mcp init --agents claude-code
```

The installer patches `~/.claude/mcp.json` and drops scaffolds under
`~/.claude/`. Re-running is idempotent.

## What gets written

```
~/.claude/
├── mcp.json                     # adds the "lighter" entry under mcpServers
├── commands/lighter-*.md        # 5 slash commands
├── agents/lighter-trader.md     # sub-agent definition
└── hooks/post-tool-call.sh      # logs each lighter_(live|funds)_* call
```

The `SKILL.md` agent contract lives at
`adapters/claude/skill/SKILL.md` in the source repo — drop it under
`~/.claude/skills/lighter/` if you want it auto-attached on every
Claude Code session.

## mcp.json entry

```json
{
  "mcpServers": {
    "lighter": {
      "command": "lighter-mcp",
      "args": ["stdio"],
      "env": {
        "LIGHTER_MCP_CONFIG": "/Users/you/.lighter/lighter-mcp/config.toml"
      }
    }
  }
}
```

`LIGHTER_MCP_CONFIG` is the only env var the server needs — kit path,
mode, and safety caps all live inside the TOML.

## Reload

Restart `claude` (Ctrl-D the current session and re-run).

## Verifying

```
> /lighter-status
```

Or simply ask:

> What mode is the Lighter MCP server in, and how much daily notional do I have left?

The agent should call `lighter_safety_status` and reply.

## Notes

- The `lighter-trader` sub-agent is invoked with `@lighter-trader` and
  has its own narrow tool budget — it cannot edit your source files.
- Hooks land in `~/.claude/hooks/post-tool-call.sh`. They are
  best-effort; the MCP server's own gates remain the authoritative
  safety boundary.
