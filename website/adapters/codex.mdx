The Codex adapter ships as a **Codex plugin** that registers the MCP
server, the SKILL, the slash commands, the sub-agent, and the
post-trade hook.

## Install

```bash
lighter-mcp init --agents codex
```

The installer lays the plugin out at `~/.codex/plugins/lighter/`,
generates a `.mcp.json` pointing at your installed `lighter-mcp`
binary with `LIGHTER_MCP_CONFIG` pre-set, and runs `lighter-mcp doctor`.
Codex picks the plugin up at next launch.

## What gets written

```
~/.codex/plugins/lighter/
├── plugin.json                  # plugin manifest
├── .mcp.json                    # generated server config (paths filled in)
├── skills/lighter/SKILL.md      # agent-facing safety contract
├── commands/lighter-*.md        # 5 slash commands (symlinks)
├── agents/lighter-trader.md     # sub-agent (symlink)
└── hooks/after-lighter-trade.sh # post-tool-call hook (symlink)
```

## plugin.json sketch

```json
{
  "name": "lighter-mcp",
  "version": "0.1.0",
  "description": "Lighter trading toolkit",
  "mcp": "./.mcp.json",
  "skills": ["./skills/lighter/SKILL.md"],
  "commands": ["./commands"],
  "agents": ["./agents/lighter-trader.md"],
  "hooks": { "postToolCall": "./hooks/after-lighter-trade.sh" }
}
```

The generated `.mcp.json` looks like:

```json
{
  "mcpServers": {
    "lighter": {
      "command": "/Users/you/.local/bin/lighter-mcp",
      "args": ["stdio"],
      "env": {
        "LIGHTER_MCP_CONFIG": "/Users/you/.lighter/lighter-mcp/config.toml"
      }
    }
  }
}
```

## Reload

```bash
codex mcp restart
```

## Verifying

```
> /lighter-status
```

Or invoke the sub-agent:

```
> @lighter-trader give me a paper trading session for today
```

## Notes

- Codex's MCP integration requires **Codex CLI ≥ 0.30**. Earlier
  versions don't load plugin-defined MCP servers.
- The `lighter-trader` sub-agent has the same narrow tool budget as in
  the other adapters — it cannot edit source files.
- `codex mcp logs lighter` is the fastest way to debug a missing tool.
